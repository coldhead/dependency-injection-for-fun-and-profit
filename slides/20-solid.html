SOLID Design Principles

  • the building blocks of OO design patterns
  • if you **only** follow these, you will write programs that are easier to understand
This is OOP 101, but not necessarily JS 101. If you're experience is just using jQuery to enhance a page, SOLID isn't required knowledge

S - Single Responsibility

O - Open / Closed

L - Liskov Substitution

I - Interface Segregation

D - Dependency Inversion

a class should have only a single responsibility.

software entities … should be open for extension, but closed for modification.

objects in a program should be replaceable with instances of their subtypes without altering the correctness of that program

many client-specific interfaces are better than one general-purpose interface.

one should “Depend upon Abstractions. Do not depend upon concretions.
